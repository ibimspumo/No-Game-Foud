# Vision & Core Concept

## The Elevator Pitch

**You are a single pixel. Your purpose: convert everything into pixels. Starting from an 8x8 canvas, you will consume the gallery, the computer, the room, the house, the city, the planet, the solar system, the galaxy, the universe, the multiverse, and finally... yourself.**

## What Makes This Different

| Other Idle Games | Pixel Singularity |
|------------------|-------------------|
| Numbers go up | Numbers go up, then the MEANING changes |
| Reset for bonuses | Reset to discover new PHASES OF EXISTENCE |
| Upgrades = stronger | Upgrades = fundamentally different gameplay |
| One tone throughout | Tone shifts: cute → epic → existential → horror → transcendent |
| You control a thing | You ARE the thing, then you BECOME everything |

## The Core Loop

```
MICRO LOOP (Seconds):
Click/Generate → Resources → Buy Upgrade → More Resources

MACRO LOOP (Hours):
Complete Phase → Unlock New Mechanics → Enter New Scale → Repeat

META LOOP (Days/Weeks):
Complete Run → Rebirth → Permanent Upgrades → New Content → Repeat

TRANSCENDENT LOOP (Weeks/Months):
Unlock All Endings → True Ending → ??? → The game changes fundamentally
```

## Design Pillars

1. **ESCALATION** - Every phase should feel 10x bigger than the last
2. **EMOTION** - The player should FEEL something at key moments
3. **PHILOSOPHY** - Deep questions disguised as a clicking game
4. **SURPRISE** - Every phase introduces something unexpected
5. **AGENCY** - Player choices matter and persist

## PHASE 1: THE PIXEL

### Subtitle: "In the beginning, there was nothing. Then, there was one."

### Duration: 2-5 minutes

### The Experience

The screen is white. Completely white. Nothing happens.

After 3 seconds of nothing, a single black pixel appears in the center.

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│                                                         │
│                                                         │
│                                                         │
│                                                         │
│                           ■                             │
│                                                         │
│                                                         │
│                                                         │
│                                                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

No instructions. No UI. Just the pixel.

When the player clicks it, a soft "pop" sound. The number "1" appears briefly.

The pixel pulses slightly. Inviting another click.

### Mechanics

- **Click** = +1 Pixel
- **Nothing else.** No upgrades, no automation.
- The pixel grows slightly larger with each click (visual only)
- At certain thresholds (10, 50, 100), the pixel changes color briefly

### UI Elements

- Just the pixel
- A tiny counter appears at 10 pixels (very subtle, bottom corner)
- No menu, no settings, no anything

### Transition to Phase 2

At **64 pixels**, the screen flashes white.

Text appears:

```
"You have enough to exist."

"Do you want to... become?"

[Yes]
```

There is no "No" option. There never was a choice.

### Narrative/Logs

```
[LOG 001]
A single point of darkness in infinite light.
Is this existence? Or merely potential?

[LOG 002]
Each click feels like a heartbeat.
Whose heart? Mine? Or...?

[LOG 003]
I am 64 now. I am ready.
Ready for what?
```

### Emotional Goal

**Wonder + Simplicity.** The player should feel like they discovered something.

### New Mechanics

- **Colors** appear randomly (1% chance per click)
- **First Upgrades:**
  - Pixel Generator (10 Pixels/click → 1 Pixel/second)
  - Color Attractor (increases color chance)
- Clicking now fills the canvas automatically (production)
- Canvas fills from top-left to bottom-right

### The First Choice

At 50% canvas filled, a prompt:

```
"The canvas could be filled with order... or chaos."

[ORDER] - Pixels fill in straight lines
[CHAOS] - Pixels appear randomly

This choice affects all future canvases.
```

**Order path:** More predictable, easier to optimize
**Chaos path:** More interesting patterns, occasional "glitches" that give bonuses

### UI Elements

- 8x8 canvas (large, centered)
- Pixel counter
- Simple upgrade panel (2-3 upgrades)
- First "settings" button appears

### Transition to Phase 3

When the canvas is 100% filled:

```
"The canvas is complete."

"But a single work of art is just the beginning."

"What if there were... more?"

The canvas shrinks. It becomes a thumbnail.

Welcome to the Gallery.
```

### The Fourth Wall Cracks

This is the first moment the game acknowledges it's on the player's computer.

The "screen" in the game resembles a generic computer desktop, but with unsettling details:

- The browser has tabs that look suspiciously familiar
- There's a folder called "Personal"
- The wallpaper is... pixelating?

### New Mechanics

- **Conversion:** Instead of "filling," you now "convert"
- **Programs:** Each program on the desktop is a mini-challenge
  - Browser: Infinite scrolling content to convert
  - Files: Nested folders with pixel treasures
  - Games: Other "games" that become absorbed
- **Detection Risk:** Convert too fast = "antivirus" notices (penalty)

### Programs to Convert

| Program | Difficulty | Reward | Special |
|---------|------------|--------|---------|
| Browser | Easy | High pixels | Infinite tabs = infinite content |
| Documents | Medium | Rare colors | "Personal" folder = story content |
| Photos | Medium | Memory Fragments | Each photo becomes a canvas |
| Games | Hard | Game Essence | Other games fight back |
| System32 | Very Hard | Core Access | Unlocks Phase 5 |

### New Resource: Memory Fragments

Photos and documents contain "memories." These are story pieces about the "user" (not the player, but a fictional user).

### New Mechanics

- **Physical Objects:** Each object is a mini-conversion project
- **Weight:** Physical objects have "weight" (slower to convert)
- **Sentimental Value:** Some objects resist more (photos, gifts)
- **The Occupant:** There are signs someone lives here (but they're never seen... yet)

# Visual Design & Technical Implementation

## Design Philosophy: Thematic Abstraction

The visual representation evolves WITH the narrative. As the scale grows beyond human comprehension, the visuals become intentionally more abstract.

**This is not a limitation – it's a feature.**

```
"You cannot see a galaxy."
"You can only imagine one."
"The less I show you, the more you must feel."
```

### The Core Principle

| Scale | Human Ability | Visual Approach |
|-------|---------------|-----------------|
| Pixel | Can see | Show everything |
| Room | Can understand | Show simplified |
| City | Can imagine | Show schematic |
| Planet | Cannot grasp | Show abstract |
| Galaxy | Beyond comprehension | Show minimal |
| Universe | Incomprehensible | Show nothing |

### Why No Game Engine?

1. **Thematic:** A game about pixels should BE pixels
2. **Accessible:** Runs in any browser
3. **Lightweight:** No downloads, instant play
4. **Meta:** The simplicity IS the message

<script lang="ts">
  import { onMount } from 'svelte';
  
  export let text: string;
  export let speed: number = 50;
  export let delay: number = 0;
  
  let displayedText = '';
  let currentIndex = 0;

      typeNext();
    }, delay);
  });

      displayedText += text[currentIndex];
      currentIndex++;
      setTimeout(typeNext, speed);

  }

## Performance Optimization

### For Large Grids (Phase 13+)

When displaying thousands of dots:

```svelte
<script>
  import { onMount } from 'svelte';
  
  export let starCount = 1000;
  
  let canvas: HTMLCanvasElement;
  let ctx: CanvasRenderingContext2D;
  
  onMount(() => {
    ctx = canvas.getContext('2d');
    drawStars();
  });
  
  function drawStars() {
    ctx.fillStyle = '#ffffff';
    
    for (let i = 0; i < starCount; i++) {
      const x = Math.random() * canvas.width;
      const y = Math.random() * canvas.height;
      const size = Math.random() * 2;
      
      ctx.beginPath();
      ctx.arc(x, y, size, 0, Math.PI * 2);
      ctx.fill();
    }
  }
</script>

<canvas 
  bind:this={canvas}
  width={800}
  height={600}
/>
```

## Summary: The Visual Rules

### The Golden Rules

1. **No external images** - Everything is CSS/SVG/Canvas
2. **Pixel-perfect in pixel phases** - Grids are sacred
3. **Abstract when vast** - Galaxies are dots, universes are empty
4. **Text is visual** - Typography carries late-game
5. **Color is meaning** - Palette tells the story
6. **Animation is emotion** - Speed reflects phase mood

### The Technical Stack

```
Framework:     Svelte 5 (Runes)
Styling:       CSS (no framework needed)
State:         Svelte Stores
Graphics:      CSS + Canvas API (for performance)
Persistence:   LocalStorage
Build:         Vite / SvelteKit
Deployment:    Static (Vercel/Netlify)
```

### What We DON'T Need

- ❌ Game engine (Phaser, PixiJS)
- ❌ Image assets
- ❌ Icon library (we make our own)
- ❌ Animation library (CSS is enough)
- ❌ Complex state management (Svelte stores are fine)
- ❌ Backend (pure client-side)
